# Защищённый кабинет Panora — подключение

Эта версия хранит рестораны, индивидуальные цены, заказы, накладные, оплаты,
задолженность и склад в Supabase. Публичный сайт без входа не получает ни одной
персональной цены.

## 1. Создать проект

1. Создайте новый проект на `supabase.com`.
2. Откройте **SQL Editor**, вставьте содержимое `schema.sql` и нажмите **Run**.
3. В **Authentication → Users** создайте пользователя-администратора.
4. Скопируйте UUID пользователя и выполните последнюю bootstrap-команду из
   `schema.sql`, подставив UUID.

## 2. Создать ресторан и вход

1. Добавьте запись в `restaurants`.
2. В **Authentication → Users** пригласите пользователя ресторана по email.
3. Добавьте запись в `profiles`: `user_id` из Authentication, `restaurant_id`
   созданного ресторана, роль `restaurant`.
4. В `restaurant_prices` задайте цену за **1 штуку** для каждого товара.

Цена другого ресторана недоступна на уровне базы данных благодаря RLS, даже если
кто-то вручную изменит запрос в браузере.

## 3. Правила количества и денег

- Основная единица склада и документов — целая штука (`quantity_pieces`).
- Заказы, склад и документы ведутся только в штуках; короб не ограничивает количество.
- Минимальный заказ — 12 штук. Проверку минимума выполняет серверная функция
  оформления заказа; интерфейс дублирует её для удобства.
- Цена фиксируется в строке заказа и накладной. Изменение будущей цены не меняет
  старые документы.
- Отгрузка создаёт положительную запись задолженности.
- Оплата создаёт отрицательную запись и уменьшает долг, включая частичную оплату.
- Остаток берётся из `inventory_balances`: полные короба и отдельные штуки.

## 4. План выпечки и закупка

- `production_schedule` задаёт, какой хлеб печь, дату выпечки и доставки,
  плановое количество в штуках и доступность для заказа.
- `recipe_items` хранит расход каждого ингредиента на одну штуку хлеба.
- `ingredients` хранит единицу измерения, остаток и процент страхового запаса.
- `ingredient_purchase_plan` суммирует потребность будущих выпечек, учитывает
  остаток и показывает, сколько муки и других ингредиентов нужно закупить.
- Фактические рецептуры нужно заполнить по технологическим картам пекарни.

## 5. Безопасность

- В сайт можно добавлять только **Project URL** и публичный **anon key**.
- `service_role` нельзя помещать в HTML/JavaScript, GitHub или отправлять в чат.
- Регистрация посетителей должна быть выключена; пользователей приглашает администратор.
- Все административные операции выполняются только пользователем с ролью `admin`.

## 6. Что осталось подключить

Для подключения интерфейса нужны Project URL и anon key из
**Project Settings → API**. После этого можно включить экран входа, кабинет
ресторана, административную отгрузку и печатную накладную.
